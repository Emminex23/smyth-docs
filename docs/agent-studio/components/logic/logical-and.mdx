# Logic AND

The `Logic AND` component performs logical `AND` operations within your agents.

## Definition and Rules

### Operational Rules

The component follows these rules:

| Input 1          | Input 2          | Verified   | Unverified |
|------------------|------------------|------------|------------|
| `Value`          | `Value`          | `True`     | `No Value` |
| `Value`          | `No Value/False` | `No Value` | `True`     |
| `No Value/False` | `Value`          | `No Value` | `True`     |
| `No Value/False` | `No Value/False` | `No Value` | `True`     |

> **Note:** `Verified` is `True` only if all inputs have value. `Unverified` is the inverse, activating workflows when conditions are unmet.



---

## Inputs

### Boolean Values

- **True**: Any non-empty value.
- **False**: Specifically 'false' or unassigned (must be explicitly set as Boolean).
