
# Shopify Integration

`Shopify` is a leading e-commerce platform designed to help businesses and individuals create their own online stores to sell products effectively. This platform offers a comprehensive set of tools and features that facilitate the management of products, inventory, payments, and shipping. These features are designed to be accessible to users regardless of their technical proficiency.

At `SmythOS`, we excel in providing a seamless integration with `Shopify`, enhancing its powerful capabilities with customized components specifically crafted to streamline integrations within your agents' workflows. Our integration enables a more efficient, cohesive approach to managing e-commerce operations, ensuring that you can leverage the best of `Shopify` directly within our environment.

## Get Blogs

This section provides an overview of fetching blog data from your `Shopify` store using our integration tools. Utilize this feature to retrieve and manage blog content directly within `SmythOS`.

### Settings

#### Access Token

To use the API effectively, setup is required to access and manage tokens. Follow these steps to obtain an `Access Token` from Shopify:

1. **Log in to your Admin Panel:** Visit your `Shopify` admin panel and navigate to the `Apps` section.
2. **Manage Private Apps:** Look for the `Manage private apps` option, usually at the bottom of the page. Click on it to create or view your API credentials.
3. **Create or Access Token:** If you haven't created a private app yet, click `Create new private app`. If you already have an app, you can view and copy the API credentials including the `Access Token`.
4. **Finalize and Secure Access:** Once you have your `Access Token`, return to the settings of the component. Locate the specified section designated for API integration and input your generated `Access Token`.

> **Note:** Treat the `Access Token` as sensitive data. In `SmythOS`, you can securely manage it using the `Vault` feature. For more details, please refer to [this](https://app.smythos.com/doc/#/vault/keysManagement) section in the documentation.

#### Limit 

**Default value:** `50`

The `Limit` parameter specifies the maximum number of blog entries retrieved per API call. This setting helps in managing the volume of data fetched, ensuring that performance and load times are optimized. You can adjust this setting based on your specific needs and the capabilities of your application.

<Image src="/img/shopify/screenshot1.png" alt="alt text" />

### Inputs

When configuring the component, ensure the following mandatory and optional input parameters are correctly set up:

1. **store_name `required<string>`:** The unique identifier name of your `Shopify` store, required for API requests.

<Image src="/img/shopify/screenshot2.png" alt="alt text" />

> **Note:** With the flexibility and customization in mind, `SmythOS` allows you to expand the default input list by adding new inputs using the `+` button situated in the `Inputs` section.

### Outputs

Upon successful execution of the component, anticipate the following outputs which outline the data structure returned from the server:

1. **Response:** The raw response body from the API call.
2. **Headers:** Information about the response headers returned by the API.
3. **Blogs:** An array containing blog posts retrieved based on the given inputs and settings.

<Image src="/img/shopify/screenshot3.png" alt="alt text" />

> **Note:** Similar to inputs, you can add new outputs by clicking on the `+` button in the `Outputs` section. This feature can come in handy when you aim to extract specific values from the `Response`. Please refer [here](https://app.smythos.com/doc/#/components/APICall?id=example-without-authentication) to see a practical example.

## List Articles

This section provides guidelines on how to retrieve a list of articles from your `Shopify` store blogs through the `SmythOS`. Utilize these instructions to effectively access, filter, and manage content across your blog articles.

### Settings

#### Access Token

To use the API effectively, setup is required to access and manage tokens. Follow these steps to obtain an `Access Token` from Shopify:

1. **Log in to your Admin Panel:** Visit your `Shopify` admin panel and navigate to the `Apps` section.
2. **Manage Private Apps:** Look for the `Manage private apps` option, usually at the bottom of the page. Click on it to create or view your API credentials.
3. **Create or Access Token:** If you haven't created a private app yet, click `Create new private app`. If you already have an app, you can view and copy the API credentials including the `Access Token`.
4. **Finalize and Secure Access:** Once you have your `Access Token`, return to the settings of the component. Locate the specified section designated for API integration and input your generated `Access Token`.

> **Note:** Treat the `Access Token` as sensitive data. In `SmythOS`, you can securely manage it using the `Vault` feature. For more details, please refer to [this](https://app.smythos.com/doc/#/vault/keysManagement) section in the documentation.

#### Limit

**Default Value:** `50`

Defines the maximum number of articles to retrieve in one API call. Adjust this setting according to your data management needs.

#### Published Status

Select the publication status of the articles you wish to retrieve:

- `any`: All articles regardless of their publication status.
- `published`: Only articles that are currently published.
- `unpublished`: Only articles that are not publicly available.

<Image src="/img/shopify/screenshot7.png" alt="alt text" />

### Inputs

Configure the following parameters to tailor the API request to your needs:

1. **store_name `required<string>`:** Your Shopify store's unique identifier.
2. **blog_id `required<string>`:** The specific blog's ID within your Shopify store.
3. **author `optional<string>`:** Filter articles by author.
4. **created_at_max `optional<date>`:** Upper limit for article creation date (ISO 8601 format).
5. **created_at_min `optional<date>`:** Lower limit for article creation date (ISO 8601 format).
6. **fields `optional<string>`:** Comma-separated list specifying which fields to return in the response.
7. **handle `optional<string>`:** Specific article handle.
8. **published_at_max `optional<date>`:** Upper limit for article publication date (ISO 8601 format).
9. **published_at_min `optional<date>`:** Lower limit for article publication date (ISO 8601 format).
10. **since_id `optional<string>`:** Filters results to articles after the specified ID.
11. **tag `optional<string>`:** Filter articles by tag.
12. **updated_at_max `optional<date>`:** Upper limit for the last update date (ISO 8601 format).
13. **updated_at_min `optional<date>`:** Lower limit for the last update date (ISO 8601 format).

<Image src="/img/shopify/screenshot8.png" alt="alt text" />

> **Note:** With the flexibility and customization in mind, `SmythOS` allows you to expand the default input list by adding new inputs using the `+` button situated in the `Inputs` section.

### Outputs

Upon a successful execution, expect the following data in response:

1. **Response:** The complete raw data retrieved from the server.
2. **Headers:** Detailed information about the response headers.
3. **articles:** An array of article data structured according to your input parameters.

<Image src="/img/shopify/screenshot9.png" alt="alt text" />

> **Note:** Similar to inputs, you can add new outputs by clicking on the `+` button in the `Outputs` section.
