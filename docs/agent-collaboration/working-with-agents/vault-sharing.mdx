# Vault Sharing

The SmythOS Vault provides centralized, secure storage for all API keys and credentials. Team members can safely access and manage authentication tokens while maintaining security best practices.

## Available Integrations

### Built-in AI Providers

SmythOS offers pre-configured access to leading AI providers:

| Provider | Status | Documentation |
|----------|---------|---------------|
| OpenAI | Available by default | [API Reference](https://platform.openai.com/) |
| Anthropic | Available by default | [API Reference](https://docs.anthropic.com/) |
| Google AI | Available by default | [API Reference](https://console.developers.google.com) |
| Perplexity | Available by default | [API Reference](https://docs.perplexity.ai/) |

> **Note:** Using built-in providers consumes credits from your SmythOS plan

### Enterprise Model Benefits
- • **Control**: Manage model parameters and configurations directly in your systems
- • **Security**: Enterprise-grade data protection
- • **Scalability**: Handle high-volume processing needs

### Setting Up Enterprise Models

1. **Initial Setup** 📌
   - Navigate to Vault
   - Click "Add Enterprise Model"
   - Select provider (Bedrock or Vertex AI)

2. **Basic Configuration** 📝
   - Enter unique model name
   - Select features:
     - • Text completion (currently available)
     - • More features coming soon
   - Choose provider platform

3. **Provider-Specific Setup** ⚙️
   Details will vary depending on the provider you choose.
   **For Vertex AI:**
   - Required credentials:
     - • Google Service Account
     - • Project ID
     - • Region selection

   **For Amazon Bedrock:**
   - Required credentials:
     - • AWS Access Key ID
     - • AWS Secret Access Key
     - • Region selection

4. **Advanced Configuration** 🎯
   - Context window size
   - Maximum output tokens
   - Temperature settings
   - Top-p sampling
   - Response formats

5. **Deployment** 🚀
   - Click **Create**. The model will be added to your list of available models.
   - Verify configuration and test model response to ensure it is properly configured.

> **Note**: Some configuration options may vary depending on the selected model and provider.

## Managing API Keys
In the API Keys section, you can add any number of keys by providing:
   - • **Key**: Your API token (max 10,000 chars)
   - • **Name**: Unique identifier
   - • **Scope:** Choose from *All, API Call, Hugging Face, Zapier Action*.

### Scope Types
- • **All** - Unrestricted access
- • **API Call** - REST endpoint access
- • **Hugging Face** - Model hub access
- • **Zapier Action** - Automation integration

### Key Operations
- • Copy - Quick clipboard access
- • Update - Modify existing keys
- • Delete - Remove unused keys

## Managing Keys in Agents

### API Call

- **URL GET Parameters:** Use keys directly in the `URL` field.
- **Headers:** Use keys in request headers.
- **Body:** Depend on `Body` type selected, apply keys as needed.
- **OAuth:** Use keys for Client ID and Client Secret.

### Hugging Face

- **Access Token:** Apply keys in the `Access Token` field.

### Zapier Action
- **Automatic Retrieval:** Keys with Zapier scope will auto-fetch corresponding actions. Please refer to this [section](/docs/agent-studio/integrations/tools/zapier-integration).
