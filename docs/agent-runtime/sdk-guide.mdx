# Using the Smyth Runtime SDK

<InfoCallout title="Why use the SDK?">
The Smyth SDK is the fastest way to build intelligent agents programmatically. It gives you full control of agent logic, components, and workflows while abstracting away the runtime complexity.
</InfoCallout>

<Spacer size="md" />

## What the SDK does

The SDK is a developer layer built on top of the [Smyth Runtime Environment](/docs/agent-runtime/overview). It provides type-safe APIs to:

- Create and configure AI agents
- Add custom skills and tools
- Build workflows using modular [components](/docs/agent-runtime/components)
- Manage conversations with context
- Scale from local development to production environments

<Spacer size="md" />

## Creating an agent

You can define an agent by using `new SDK.Agent()`, providing a name, model, and default behavior. This establishes your agent’s base persona.

```typescript
const agent = new SDK.Agent({
  name: 'ResearchBot',
  model: 'gpt-4',
  behavior: 'You provide high-quality, cited information on scientific topics.',
});
```

<TipCallout title="Use case fit">
Use this approach when you want a reusable agent that acts consistently across different prompts and flows.
</TipCallout>

<Spacer size="md" />

## Adding custom skills

Skills are functions that can be called by your agent, or exposed as tools to the underlying LLM. Use `agent.addSkill()` to define what each skill does.

```typescript
agent.addSkill({
  name: 'getSummary',
  description: 'Summarizes a user input',
  ai_exposed: true,
  process: async (input) => {
    return { summary: `Summary: ${input}` };
  },
});
```

<TipCallout title="Expose only what you need">
Set `ai_exposed: true` only for tools you want the agent to use in its reasoning process. Internal helper functions can omit this flag.
</TipCallout>

<Spacer size="md" />

## Prompting the agent

Use `agent.prompt()` for a single interaction. This method does not retain any memory of previous messages.

```typescript
const response = await agent.prompt('Explain the Doppler effect.');
console.log(response);
```

<Spacer size="md" />

## Using chat mode

To maintain conversation history and context, use `agent.chat()`. This keeps state between each message.

```typescript
const chat = agent.chat();

await chat.send('What is general relativity?');
await chat.send('Compare it with quantum theory.');

console.log(chat.history);
```

<TipCallout title="Context matters">
Use chat mode when building assistants that require context, such as support agents or tutors.
</TipCallout>

<Spacer size="md" />

## Building workflows with components

The SDK provides access to every [SRE component](/docs/agent-runtime/components), such as LLMs, classifiers, and API calls. Components can be connected programmatically.

```typescript
const llm = SDK.GenAILLM({ model: 'gpt-4', prompt: 'Explain entropy' }, agent);
const classifier = SDK.Classifier({ classes: ['physics', 'math', 'philosophy'] }, agent);

classifier.in({ Input: llm.out.Reply });
```

<Spacer size="md" />

## Extending functionality with custom components

For custom workflow logic, extend the base `Component` class. Custom components fit directly into agent workflows.

```typescript
export class UppercaseTransform extends Component {
  async process(input, config, agent) {
    return { result: input.toUpperCase() };
  }
}
```

<Spacer size="md" />

## Wrapping external APIs as skills

You can connect to external APIs by defining a skill.

```typescript
agent.addSkill({
  name: 'getWeather',
  description: 'Returns weather info',
  ai_exposed: true,
  process: async (city) => {
    const res = await fetch(`https://api.weatherapi.com/v1/current.json?q=${city}`);
    const data = await res.json();
    return { temp: data.current.temp_c, condition: data.current.condition.text };
  },
});
```

<WarningCallout title="Async network handling">
Always wrap external API calls with error handling. The SDK does not retry failed skill executions automatically.
</WarningCallout>

## Configuring the runtime

Agents built with the SDK can run in any [runtime configuration](/docs/agent-runtime/deployment), from local development to production.

```typescript
// Local config (development)
const sre = SRE.init({
  Cache: { Connector: 'RAM' },
  Storage: { Connector: 'Local' },
  Log: { Connector: 'ConsoleLog' },
});
await sre.ready();
```

```typescript
// Production-ready config
const sre = SRE.init({
  Cache: { Connector: 'Redis', Settings: { url: 'redis://prod-cache' } },
  Storage: { Connector: 'S3', Settings: { bucket: 'agent-data' } },
  Vault: { Connector: 'HashicorpVault', Settings: { url: 'https://vault.company.com' } },
  Log: { Connector: 'SmythLog', Settings: { level: 'info' } },
});
await sre.ready();
```

<TipCallout title="Zero code changes">
Your agent and skill logic remain the same across all environments. Only the infrastructure configuration changes.
</TipCallout>

## SDK component coverage

The SDK provides type-safe wrappers for all official [SRE components](/docs/agent-runtime/components).

| Category | Examples |
| :--- | :--- |
| LLMs and AI | `GenAILLM`, `VisionLLM`, `LLMAssistant` |
| External APIs | `APICall`, `WebScrape`, `ZapierAction` |
| Data tools | `DataSourceIndexer`, `Classifier`, `JSONFilter` |
| Logic and flows | `LogicAND`, `ForEach`, `Await` |
| File and code | `FileStore`, `ServerlessCode`, `Code` |
