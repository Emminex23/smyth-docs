# Trello Integration

`Trello` stands out as a popular web-based project management application renowned for its intuitive kanban paradigm. Empowering users to create boards, lists, and cards, `Trello` facilitates efficient task organization and collaboration within teams. With its simplicity and flexibility, `Trello` caters to diverse use cases, including project management, task tracking, and team collaboration.

At `SmythOS`, we prioritize seamless integration with `Trello`, enriching its robust features with tailor-made components to streamline integration within your agents' workflow.

## Get an Action

Introducing the `Get an Action` component, designed to provide effortless access to action details within `Trello`. Whether tracking project progress, monitoring task updates, or analyzing team activity, this component facilitates seamless retrieval of crucial action data.

### Settings

To initialize and configure the `Get an Action` component, follow these simple guidelines:

1. If you haven't already, register for an account on the `Trello` platform to access its project management features.
2. Create a `Power-Up` for your `Trello` account and manage it by navigating to [this link](https://trello.com/power-ups/admin). This step is essential for enabling API access to your `Trello` account.
3. Within the `Power-Up` management section, you can generate an `API Key` and an `Access Token` for API integration. Follow the provided instructions to obtain these credentials.
4. Once you have obtained your `API Key` and `Access Token`, return to the settings of the component. Locate the specified section designated for API integration and input your generated `API Key` and `Access Token`.

> **Note:** Treat the `Access Token` and `API Key` as sensitive data. In `SmythOS`, you can securely manage it using the `Vault` feature. For more details, please refer to [this](https://app.smythos.com/doc/#/vault/keysManagement) section in the documentation.

<Image src="/img/trello/screenshot1.png" alt="alt text" />

### Inputs

To effectively leverage the capabilities of the `Get an Action` component, ensure the provision of the following input parameter:

1. **action_id `required<string>`:** Specify the unique identifier (ID) of the action you wish to retrieve details for. This ID facilitates accurate identification and retrieval of the designated action within the `Trello` platform.

<Image src="/img/trello/screenshot2.png" alt="alt text" />

> **Note:** With the flexibility and customization in mind, `SmythOS` allows you to expand the default input list by adding new inputs using the `+` button situated in the `Inputs` section.

### Outputs

Following the successful execution of the `Get an Action` component, you can expect the outputs detailed below:

1. **Response:** An object that includes extensive details about the action requested. It gathers critical information such as the action's type, the exact timestamp when the action was performed, and any associated data that is relevant to the action carried out on the `Trello` platform.
2. **Headers:** These are the HTTP headers obtained from `Trello`'s server response. They provide further metadata that is instrumental in understanding the context of the response, including response status codes which indicate the success of the operation and the content type that details the format of the data returned.
3. **actionId:** The unique identifier assigned to the specific action within `Trello`. This ID is vital for referencing the action in future queries or for integration purposes where the action's details might need to be retrieved or examined at a later time.
4. **actionData:** The actual data associated with the action that was performed. This portion of the response will often contain specifics such as the entities involved in the action (like cards, lists, or boards), any changes that were made, and metadata that provides context for the action within the `Trello` ecosystem.

<Image src="/img/trello/screenshot3.png" alt="alt text" />

> **Note:** Similar to inputs, you can add new outputs by clicking on the `+` button in the `Outputs` section. This feature can come in handy when you aim to extract specific values from the `Response`. Please refer [here](https://app.smythos.com/doc/#/components/APICall?id=example-without-authentication) to see a practical example.
