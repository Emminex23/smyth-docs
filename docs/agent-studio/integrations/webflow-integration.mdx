# Webflow Integration

The `Webflow` API allows developers to interact with their `Webflow` projects programmatically. By using this API, you can automate various aspects of your `Webflow` sites, such as managing content, form submissions, and collections, directly from your applications or workflows. This API is essential for developers who need to integrate `Webflow` with other services, build custom features, or enhance automation capabilities.

Integrating the `Webflow` API into `SmythOS` provides several benefits:

- **Centralized Management**: Manage all your `Webflow` projects and content directly from the `SmythOS` platform, ensuring a centralized location for your web development tasks.
- **Automation**: Automate repetitive tasks, such as updating content or processing form submissions, to save time and reduce manual workload.
- **Custom Integrations**: Connect `Webflow` with other tools and services you use, enabling a more seamless and integrated workflow.
- **Enhanced Capabilities**: Leverage `SmythOS`'s features, such as task management and project tracking, in conjunction with `Webflow`'s web development capabilities.

With `Webflow` integration in `SmythOS`, users can streamline their web project management and enhance overall productivity by combining the robust features of both platforms.

## Delete Collection Item

Delete an item from a collection in `Webflow`.

### Settings

#### API Token

To utilize the Webflow API, you will need an `API Token`. Follow these steps to get your API Token:

1. **Log in to Webflow**: Access your Webflow account and navigate to the dashboard.
2. **Access Account Settings**: Click on your profile icon and go to `Account Settings`.
3. **Generate a New API Token**: Look for the API section, select `Generate New Token`, and follow the prompts to create a new API token.
4. **Save Your Token**: Once generated, copy the token and store it securely as it will only be shown once.
5. **Enter Credentials in SmythOS**: Input the acquired `API Token` into the designated settings section within the component.

> **Note:** Treat the `API Token` as sensitive data. In `SmythOS`, you can securely manage it using the `Vault` feature. For more details, please refer to [this](https://app.smythos.com/doc/#/vault/keysManagement) section in the documentation.

<Image src="/img/webflow/screenshot1.png" alt="alt text" /> 

### Inputs

To effectively utilize the component for deleting collection items, provide the following inputs:

- **collection_id (`required<string>`)**: The unique identifier of the collection.
- **item_id (`required<string>`)**: The unique identifier of the item to be deleted.

<Image src="/img/webflow/screenshot2.png" alt="alt text" />

> **Note:** With the flexibility and customization in mind, `SmythOS` allows you to expand the default input list by adding new inputs using the `+` button situated in the `Inputs` section.

### Outputs

After processing the inputs, this feature generates detailed outputs, including:

1. **Response**: Contains the dataset returned from `Webflow`, including any relevant information about the deleted item.
2. **Headers**: Includes HTTP headers that provide insights into the API request-response process, such as status codes, rate limits, and other key metadata.

<Image src="/img/webflow/screenshot3.png" alt="alt text" />

> **Note:** Similar to inputs, you can add new outputs by clicking on the `+` button in the `Outputs` section. This feature can come in handy when you aim to extract specific values from the `Response`. Please refer [here](https://app.smythos.com/doc/#/components/APICall?id=example-without-authentication) to see a practical example.
