# Gmail Integration with SmythOS

> **Want to put your email on autopilot (the smart way)?** Connect Gmail to SmythOS and let your agents handle email tasks, from sending and reading to organizing and automating follow-ups.

<InfoCallout>
**TL;DR:** Set up Google OAuth 2.0 credentials, authenticate within SmythOS, and then use our suite of Gmail components to seamlessly integrate email functionalities into your agent workflows.
</InfoCallout>

<Divider />

## Why Integrate Gmail with Your Agent?

Integrating Gmail with SmythOS unlocks powerful email automation capabilities for your agents. Here is how it can help you:
- **Automate Email Sending & Receiving**: Agents can send emails, read incoming messages, and parse content, all based on your defined logic and triggers. 
- **Streamline Communication Workflows**: Trigger agent actions based on new emails, or have agents compose and send emails as part of a larger automated process.  
- **Organize Your Inbox Programmatically**: Agents can manage drafts, move messages to trash, or apply labels, helping you keep your Gmail organized without manual intervention. 
- **Enhance Data Collection**: Extract information from emails to use in other parts of your SmythOS workflows, such as updating a database or a Google Sheet. 

By connecting Gmail, you transform a core communication tool into an active, intelligent component of your SmythOS automations, saving time and enabling more sophisticated interactions.

## Prerequisites: What You Will Need to Get Started

Before you begin the integration, please ensure you have the following:

- An active **SmythOS account**. (If you are new to SmythOS, [you can sign up here](https://app.smythos.com/signup).)
- A **Google Account** with Gmail enabled that you plan to integrate.
- You will need to grant SmythOS permission to interact with your Gmail account during the authentication process.
- Access to the **Google Cloud Console** to create OAuth 2.0 credentials.

<Divider />

## Step by Step: Connecting Your Gmail Account

To interact with Gmail through SmythOS, you first need to set up Google OAuth 2.0 credentials and then authenticate within SmythOS.

### Step 1: Configure Google OAuth 2.0 Credentials

For a visual guide on this setup, you can refer to typical Google OAuth setup processes. (The original document mentioned a video for this section; if you have one, its link can be placed here.)

1.  **Visit Google Cloud Console**: Access the [Google Cloud Console](https://console.cloud.google.com/).
2.  **Create or Select Your Project**: Choose an existing project or create a new one if you have not already.
3.  **Enable the Gmail API**: Navigate to the **API & Services** > **Library**, search for **Gmail API**, and enable it for your chosen project.
4.  **Configure OAuth Consent Screen**: If you have not already, go to **API & Services** > **OAuth consent screen**. Configure it as an "External" user type (unless you are a Google Workspace admin and this is for internal users only). Add necessary scopes, typically including `https://www.googleapis.com/auth/gmail.modify` or other scopes relevant to the actions your agents will perform.
5.  **Generate OAuth 2.0 Credentials**: Go to **API & Services** > **Credentials**. Click **Create Credentials** and select **OAuth client ID**.
    * Choose "Web application" as the Application type.
    * Add an "Authorized redirect URI": `https://app.smythos.com/oauth/google/callback` (or your specific SmythOS callback URL if different). This ensures proper redirection post authentication.
6.  **Retrieve Client ID and Client Secret**: After creating the OAuth client ID, Google will provide you with a **Client ID** and a **Client Secret**. Copy these values securely.

<WarningCallout>
    **Treat your `Client ID` and `Client Secret` as sensitive data.** In SmythOS, you can securely manage these credentials using the **Vault** feature. For more details, please refer to the [Vault Keys Management](https://app.smythos.com/doc/#/vault/keysManagement) section in our documentation.
</WarningCallout>

### Step 2: Authenticate in SmythOS

1.  In SmythOS, when adding a Gmail component or configuring the Gmail integration globally (if applicable), you will be prompted to enter your **Client ID** and **Client Secret**.
2.  Input these credentials into the respective fields.
3.  Click the **Authenticate** button to establish the connection. This will typically redirect you to Google for authorization.
4.  Grant the requested permissions to SmythOS.
5.  Upon successful authorization, you will be redirected back to SmythOS. **Connection Confirmed!** Your Gmail account is now connected.

## Troubleshooting Common Issues

Encountering an issue with your Gmail integration? Here are solutions to some common problems:

* **Error: `invalid_grant` or Authentication Errors**
    * **Possible Cause(s):** Incorrect Client ID/Secret, misconfigured OAuth consent screen or redirect URIs, revoked access, or expired refresh tokens.
    * **Solution(s):**
        1.  Double check your Client ID and Client Secret in the Google Cloud Console and SmythOS.
        2.  Ensure the redirect URI (`https://app.smythos.com/oauth/google/callback` or your specific URL) is correctly listed in your Google OAuth client ID settings.
        3.  Verify the Gmail API is enabled in your Google Cloud Project.
        4.  Try re authenticating the connection in SmythOS. You may need to go through the Google consent screen again.
        5.  Ensure the scopes configured on your OAuth consent screen match the operations your agent is trying to perform.

* **Error: `Requested entity was not found` (or a `404` error for specific IDs)**
    * **Possible Cause(s):** The provided `draftId` or `messageId` does not exist, was mistyped, or has already been deleted.
    * **Solution(s):** Verify the ID is correct and still exists in your Gmail account. Use "List" components to confirm IDs if unsure.

* **Error: `PERMISSION_DENIED` (or a `403` error)**
    * **Possible Cause(s):** The authenticated user does not have sufficient permissions for the requested operation on the specific draft/message, or the necessary API scopes were not granted during OAuth.
    * **Solution(s):**
        1.  Ensure the Google account used for authentication has the necessary rights (e.g., to modify or delete a draft/message).
        2.  Check the scopes granted to your application in the Google Cloud Console OAuth consent screen settings. You might need to add scopes like `gmail.modify`, `gmail.readonly`, `gmail.compose`, etc., and then have users re-authorize.

* **Exceeding API Rate Limits or Other Google Quotas**
    * **Possible Cause(s):** Gmail imposes limits on API call frequency and volume. High frequency operations (e.g., listing or sending many messages rapidly) can hit these limits.
    * **Solution(s):** Implement rate limiting or introduce strategic delays in your agent workflows if performing bulk operations. Refer to Google's official Gmail API documentation for current usage limits. Consider using batch operations if available for your use case.
