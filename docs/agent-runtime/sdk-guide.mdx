# Using the Smyth SDK

<InfoCallout title="Why use the SDK">
The Smyth SDK is the fastest way to build intelligent agents programmatically. It gives you full control of agent logic, components, and workflows while abstracting away the runtime complexity.
</InfoCallout>

<Spacer size="md" />

## What the SDK does

The SDK is a developer layer built on top of the [Smyth Runtime Environment](/docs/agent-runtime/overview). It provides type-safe APIs to:

- Create and configure AI agents
- Add custom skills
- Build workflows using modular components
- Manage conversations with context
- Scale across development and production environments

<Spacer size="md" />

## Creating an agent

You start with a basic agent definition that includes a name, model, and default behavior. This sets the base persona for your agent.

```ts
const agent = new SDK.Agent({
  name: 'ResearchBot',
  model: 'gpt-4',
  behavior: 'You provide high-quality, cited information on scientific topics.',
});
```

<TipCallout title="Use case fit">
Use this approach when you want a reusable agent that can act consistently across different prompts and flows.
</TipCallout>

<Spacer size="md" />

## Adding custom skills

Skills are code functions your agent can call internally or expose to the LLM. You define what it does, when it runs, and whether it's available to the model at runtime.

```ts

agent.addSkill({
  name: 'getSummary',
  description: 'Summarizes a user input',
  ai_exposed: true,
  process: async (input) => {
    return { summary: `Summary: ${input}` };
  },
});

```

<TipCallout title="Expose only what you need">
Set `ai_exposed: true` only for tools you want the agent to use in reasoning. Internal tools can skip this flag.
</TipCallout>

<Spacer size="md" />

## Prompting the agent

Use `agent.prompt()` for a one-shot interaction. This is great for tools, APIs, or functions that don’t need memory of previous messages.

```ts

const response = await agent.prompt('Explain the Doppler effect.');
console.log(response);

```

<Spacer size="md" />

## Using chat mode

To maintain conversational history, switch to chat mode. It keeps state between messages.

```ts

const chat = agent.chat();

await chat.send('What is general relativity?');
await chat.send('Compare it with quantum theory.');

console.log(chat.history);

```

<TipCallout title="Context matters">
Use chat when building assistants that need context, such as support agents or tutors.
</TipCallout>

<Spacer size="md" />

## Building workflows with components

The SDK gives you access to all [SRE components](/docs/agent-runtime/components), like LLMs, classifiers, API calls, and logic blocks. These can be connected programmatically.

```ts

const llm = SDK.GenAILLM({ model: 'gpt-4', prompt: 'Explain entropy' }, agent);
const classifier = SDK.Classifier({ classes: ['physics', 'math', 'philosophy'] }, agent);

classifier.in({ Input: llm.out.Reply });

```

<Spacer size="md" />

## Extending functionality

If you need custom logic, write your own component by extending the `Component` class. It fits naturally into the agent's flow.

```ts

export class UppercaseTransform extends Component {
  async process(input, config, agent) {
    return { result: input.toUpperCase() };
  }
}

```

<Spacer size="md" />

## Wrapping external APIs as skills

Skills can connect to outside services or run custom workflows.

```ts

agent.addSkill({
  name: 'getWeather',
  description: 'Returns weather info',
  ai_exposed: true,
  process: async (city) => {
    const res = await fetch(`https://api.weatherapi.com/v1/current.json?q=${city}`);
    const data = await res.json();
    return { temp: data.current.temp_c, condition: data.current.condition.text };
  },
});

```

<WarningCallout title="Async network handling">
Always wrap external calls with proper error handling. The SDK does not retry failed skill executions automatically.
</WarningCallout>

<Spacer size="md" />

## Configuring the runtime

Your agent code can run against any [runtime configuration](/docs/agent-runtime/deployment), from local dev to production setups.

```ts
// Local config (development)
const sre = SRE.init({
  Cache: { Connector: 'RAM' },
  Storage: { Connector: 'Local' },
  Log: { Connector: 'ConsoleLog' },
});
await sre.ready();

```

```ts
// Production-ready config
const sre = SRE.init({
  Cache: { Connector: 'Redis', Settings: { url: 'redis://prod-cache' } },
  Storage: { Connector: 'S3', Settings: { bucket: 'agent-data' } },
  Vault: { Connector: 'HashicorpVault', Settings: { url: 'https://vault.company.com' } },
  Log: { Connector: 'SmythLog', Settings: { level: 'info' } },
});
await sre.ready();

```

<TipCallout title="Zero code changes">
Your agent and skill logic stays the same across environments. Only your infrastructure setup changes.
</TipCallout>
