
# Stability AI Integration

`Stability AI` is a prominent company in the field of artificial intelligence, known for its contributions to open-source generative AI tools.

At `SmythOS`, we prioritize seamless integration with `Stability AI`, enriching its robust features with tailor-made components to streamline integration within your agents' workflow.

## Text-to-Image SD 1.6

Utilize Stable Diffusion 1.6 for text to image generation with Stability AI.

### Settings

#### API Key

To initialize and configure the component, follow these simple guidelines:

1. If you haven't already, register for an account on the `Stability AI` platform to access its image generation features.
2. Navigate to the [API Keys](https://platform.stability.ai/account/keys) section in your `Stability AI` dashboard.
3. Generate your `API Key` by following the provided instructions.
4. Return to the settings of the component and input your generated `API Key` into the specified section designated for API integration.

> **Note:** Treat the `API Key` as sensitive data. In `SmythOS`, you can securely manage it using the `Vault` feature. For more details, please refer to [this](https://app.smythos.com/doc/#/vault/keysManagement) section in the documentation.

## Inpaint

### Settings

1. If you haven't already, register for an account on the `Stability AI` platform to access its image generation features.
2. Navigate to the [API Keys](https://platform.stability.ai/account/keys) section in your `Stability AI` dashboard.

4. Return to the settings of the component and input your generated `API Key` into the specified section designated for API integration.

<Image src="/img/stabilityai/screenshot4.png" alt="alt text" />

### Inputs

Prepare your image for processing with these input parameters:

1. **image_binary `required<binary>`**: Upload the primary image for modification.
2. **prompt `required<string>`**: Describe your desired changes or additions to the image.
3. **mask_image_binary `optional<binary>`**: Provide a mask image to specify areas for inpainting (modifies image based on pixel intensity).
4. **negative_prompt `optional<string>`**: Define elements to exclude from the final image. Defaults to a comprehensive negative keyword list.
5. **seed `optional<integer>`**: Set a seed for reproducibility. Default is `0`.
6. **output_format `optional<string>`**: Specify the output image format. Default is `png`.
