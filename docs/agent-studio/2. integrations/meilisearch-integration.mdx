
# Meilisearch Integration

`Meilisearch` is a company that develops an open-source search engine that is fast, relevant, and easy to set up. `Meilisearch` provides developers with the tools to integrate powerful search capabilities into their applications quickly.

At `SmythOS`, we've seamlessly integrated with `Meilisearch` to bring you custom components that make incorporating their advanced services into your projects a breeze.

## Full Text Search

The `Full Text Search` component empowers users to search for documents matching a specific query in the given index.

### Settings

To kickstart this component, follow these initial steps to obtain your `Master Key`:

1. Create a `Meilisearch` account if you don't already have one.
2. Navigate to your `Settings` and acquire your `Master Key`.
3. Input this `Master Key` into the specified section in the component's settings.

> **Note:** Treat the `Master Key` as sensitive data. In `SmythOS`, you can securely manage it using the `Vault` feature. For more details, please refer to [this](https://app.smythos.com/doc/#/vault/keysManagement) section in the documentation.

<Image src="/img/meilisearch/screenshot1.png" alt="alt text" />

### Inputs

To effectively utilize this component, you need to provide the following inputs:

1. **url `required<string>`:** The base URL of your `Meilisearch` instance. Replace this with the actual URL where your `Meilisearch` instance is hosted.
2. **index_uid `required<string>`:** The unique identifier of the index you want to search in.
3. **query `required<integer>`:** The search query.

<Image src="/img/meilisearch/screenshot2.png" alt="alt text" />

> **Note:** With the flexibility and customization in mind, `SmythOS` allows you to expand the default input list by adding new inputs using the `+` button situated in the `Inputs` section.

### Outputs

After running this component, the following default outputs are generated:

1. **Response:** This output is an object that provides detailed information, which includes the search results. It serves as a comprehensive overview of the generated output post-execution.
2. **Headers:** The HTTP Headers within the response from the `Meilisearch` server are included here. These headers convey additional data such as status codes and metadata related to the response.
3. **results:** This part of the output is dedicated to presenting the results derived from the search query. It typically consists of an array or a list of items that match the search criteria.

<Image src="/img/meilisearch/screenshot3.png" alt="alt text" />

> **Note:** Similar to inputs, you can add new outputs by clicking on the `+` button in the `Outputs` section. This feature can come in handy when you aim to extract specific values from the `Response`. Please refer [here](https://app.smythos.com/doc/#/components/APICall?id=example-without-authentication) to see a practical example.
