# Use Google Gemini with SmythOS

> Connect to Google's powerful Gemini LLM to add text generation, chat completions, and AI reasoning to your SmythOS agent workflows.

<InfoCallout>
**TL;DR:** Add your Gemini API Key, choose the right model, and plug into your workflow with the `API Call` component.
</InfoCallout>

<Divider />

## Why Google Gemini?

Google Gemini offers a family of large language models that support a range of use cases including:

- Text generation and completion
- Code explanation
- Content rewriting and summarization
- Reasoning, planning, and structured output

You can use Gemini directly within your SmythOS agents via simple API integration, no special SDKs needed.

<Divider />

## Prerequisites

Before integrating Gemini:

- Create a [Google AI Studio](https://aistudio.google.com/) account
- Generate a Gemini API Key from [Google AI Studio](https://makersuite.google.com/app/apikey)
- Store this key securely in your [Vault → API Keys](/docs/agent-collaboration/working-with-agents/vault-sharing)

<Divider />

## Step-by-Step: Add Gemini to Your Agent

### Step 1: Add the API Call Component

1. In your agent, click **+ Add Component**
2. Choose **API Call** from the list
3. Open the settings and name it something like `CallGemini`

<Divider />

### Step 2: Configure Component Settings

Set these values:

- **URL**: `https://generativelanguage.googleapis.com/v1beta/models/gemini-pro:generateContent`
- **Method**: POST
- **Headers**:

```json
{
  "Content-Type": "application/json",
  "x-goog-api-key": "${GEMINI_API_KEY}"
}
```

- Replace `${GEMINI_API_KEY}` with the variable from your Vault

<Divider />

### Step 3: Define Inputs

Example input payload:

```json
{
  "contents": [
    {
      "parts": [
        {
          "text": "Write a blog post about SmythOS integrations."
        }
      ]
    }
  ]
}
```

<InfoCallout>
You can dynamically pass user inputs into the `text` field using mapped variables or prior component outputs.
</InfoCallout>

<Divider />

### Step 4: Use Outputs

You’ll receive a JSON response with:

- `candidates[0].content.parts[0].text`: The generated text
- `promptFeedback`: Safety and model usage metadata

Use the **Output Mapping** to extract and reuse the result elsewhere in your workflow (e.g., in a Slack message, email, or UI element).

<Divider />

## Use Cases

- Dynamic blog/article drafting

# Integrate Google Gemini with SmythOS

<InfoCallout>
**TL;DR:** Add your Gemini API Key to Vault, configure an API Call component to POST to the Gemini endpoint, and handle the response.
</InfoCallout>

<Divider />

## Why Integrate Google Gemini?

Google Gemini offers powerful text generation and content creation capabilities. By connecting it to SmythOS, you can:

- Generate dynamic responses
- Summarize or rephrase content
- Build intelligent assistants
- Inject LLM power into automated workflows

<Divider />

## Prerequisites

Before getting started:

- Create a [Google AI Studio](https://aistudio.google.com/) account
- Generate your Gemini API Key
- Store your API Key securely in [Vault → API Keys](/docs/agent-collaboration/working-with-agents/vault-sharing)

<Divider />

## Step-by-Step: Connect Gemini

### Step 1: Add Gemini API Key to Vault

1. Go to your Google AI Studio account
2. Generate a new API Key
3. In SmythOS, navigate to **Vault → API Keys**
4. Add your Gemini API Key with a recognizable name (e.g., `GEMINI_API_KEY`)

<Divider />

### Step 2: Configure API Call Component

1. Add an **API Call** component to your agent canvas
2. Set the **method** to `POST`
3. Use this **URL**:

```plaintext
https://generativelanguage.googleapis.com/v1beta/models/gemini-pro:generateContent
```

4. In **Headers**, include:

```json
{
  "Content-Type": "application/json",
  "x-goog-api-key": "${GEMINI_API_KEY}"
}
```

5. In the **Body**, insert:

```json
{
  "contents": [
    {
      "parts": [
        {
          "text": "Your prompt here"
        }
      ]
    }
  ]
}
```

<Divider />

### Step 3: Parse the Response

The response contains a `candidates` array. Add a **Code** or **Transform** component to:

- Extract the generated content
- Handle errors or retries
- Use the response in follow-up actions or UI updates

<InfoCallout>
You can log outputs using [Agent Logs](/docs/agent-studio/agent-settigns/logs) for monitoring and debugging.
</InfoCallout>

<Divider />

## What’s Next?

- Use Gemini output in Slack, Discord, or WhatsApp integrations
- Connect to a webhook or use [Deploy as API](/docs/agent-deployments/deployments/deploy-as-api)
- Chain with multiple LLMs for content ranking or decision making
