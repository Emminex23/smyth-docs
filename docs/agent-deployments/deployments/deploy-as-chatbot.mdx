# Deploy as Chatbot

> **Make your Agent talk to your users on your site.** Embed your SmythOS Agent as a fully branded AI assistant with just one script.

<InfoCallout title="TL;DR">
Enable the ChatBot embodiment, configure domain + settings, and paste one script block into your site’s HTML.
</InfoCallout>

<Divider />

## Why Deploy as a Chatbot?

Your [Agent](/docs/agent-studio/build-agents/agent-overview) already has skills, workflows, and logic. Deploying it as a chatbot gives users real-time access to its abilities from any page on your site.

- No extra hosting or backend required  
- Brandable and fully configurable  
- Instant access to all Agent logic

<Divider />

## Prerequisites

Before starting, ensure:

- You’ve **deployed your Agent** (any version is fine)
- The **ChatBot embodiment** is enabled in **Agent Settings**
- Your desired domain is verified in **Subdomains**
- (Optional) You have an API key in **Vault** if using authentication

<Divider />

## Step 1: Configure Domain Access

1. Go to [Subdomains](https://app.smythos.com/domains)
2. Use a SmythOS default subdomain or add your own
3. Click **Verify** to confirm domain ownership

<InfoCallout title="Staging vs Production Domains">
Use `.stage.smyth.ai` for staging, `.pstage.smyth.ai` for production. These affect the embed URL.
</InfoCallout>

<Divider />

## Step 2: Enable the ChatBot Embodiment

1. Open your Agent in SmythOS  
2. Navigate to **Settings → Deployments**  
3. Toggle **ChatBot** to ON  

<TipCallout title="Best Practice">
Use the intro message to guide user expectations like “Hi! Ask me anything about onboarding or pricing.”
</TipCallout>
<Divider />

## Step 3: Customize Appearance and Behavior

Go to **Settings → Deployments → Chatbot Configuration** to visually customize:

- **Name** — Chatbot instance label (used in settings)
- **Intro Message** — First message shown to the user
- **Personality** — System prompt that guides how the chatbot responds
- **GPT Model** — Optional override of the Agent’s default model
- **Allowed Domains** — Comma-separated list of sites the bot can appear on
- **Icon URL** — Optional launcher icon (PNG or SVG)
- **Code Syntax Theme** — How code blocks will appear in responses
- **Message View** — CHoose compact or full chat mode
- **Attachments** — Allow users to upload files into the chat

<InfoCallout title="Customize via code" collapsible={true} defaultOpen={false}>
To further customize via code, override settings when calling `ChatBot.init()`:

```js
ChatBot.init({
  domain: 'cmb5b4js00fmjh6333d0foxjr.agent.pstage.smyth.ai',
  isChatOnly: false,
  allowAttachments: false,
  introMessage: 'Hi there! How can I help you today?',
  name: 'SiteBot101',
  personality: 'You are a helpful AI assistant for onboarding questions.',
});
```
To apply CSS styling:

```css
.human-chatbox-bubble {
  font-size: 15px;
  color: #333;
}

.bot-chatbox-bubble h1 {
  font-size: 18px;
  font-weight: bold;
  color: #1e40af;
}
```
</InfoCallout>

<WarningCallout title="Code overrides UI">
If a value is defined in both the config panel and code, the script version wins.
</WarningCallout>

<Divider />

## Step 4: Embed the Chatbot

Paste this **at the end of your site’s `<body>` tag**, ideally just before `</body>`:

```html
<script src="https://cmb5b4js00fmjh6333d0foxjr.agent.pstage.smyth.ai/static/embodiment/chatBot/chatbot-v2.js"></script>
<script>
  ChatBot.init({
    domain: 'cmb5b4js00fmjh6333d0foxjr.agent.pstage.smyth.ai',
    isChatOnly: false,
    allowAttachments: false,
    introMessage: '',
    // ... additional settings ...
  });
</script>
```
<WarningCallout title="Use the right domain">
The `domain` in your code **must match** your verified subdomain. Otherwise, the chatbot will not load.
</WarningCallout>

<Divider />

## Step 5: Confirm It's Live

After embedding:

- Visit the page where the script was added
- Your chatbot launcher should appear
- Open the chat and confirm it responds as expected

<SuccessCallout>
That’s it! Your Agent is now live as a chatbot on your website.
</SuccessCallout>
