# GoHighLevel Integration

GoHighLevel is a comprehensive platform designed to help businesses manage their marketing, sales, and customer service operations from a single dashboard. With a robust set of tools including CRM, email marketing, SMS marketing, and more, GoHighLevel aims to streamline business processes and improve overall efficiency.

The GoHighLevel API allows developers to integrate GoHighLevel's powerful features into their own applications. The API provides access to various functionalities such as contact management, lead generation, appointment scheduling, and more, making it a vital tool for businesses looking to enhance their operational capabilities.

Integrating the GoHighLevel API into SmythOS significantly enhances the platform's capabilities by:
- **Streamlining Operations:** Facilitating seamless management of marketing, sales, and customer service processes.
- **Improving Efficiency:** Automating repetitive tasks and workflows, thereby boosting productivity.
- **Centralizing Data Management:** Providing a single platform to manage all customer interactions and data.
- **Enhancing Customer Engagement:** Utilizing advanced marketing tools to engage with customers more effectively.

## Integration Settings

#### GoHighLevel Location API Key
To integrate this component, please follow these steps to obtain your GoHighLevel Location API Key:
1. **Log in to GoHighLevel:**
   - Navigate to the [GoHighLevel Dashboard](https://app.gohighlevel.com/dashboard/).
2. **Access Sub account:**
   - Select a **sub account** from the side menu.
   - From the sub account dashboard, go to **Settings**.
   - Select **Business Info** from the side menu.
   - Select **API Keys** from the side menu.
   - Click on **Generate API Key** if you do not already have one. 
3. **Copy the API Key:**
   - Once generated, copy your **Location API Key**.

#### GoHighLevel Agency API key
To integrate this component, please follow these steps to obtain your GoHighLevel Agency API Key:
1. **Log in to GoHighLevel:**
   - Navigate to the [GoHighLevel Dashboard](https://app.gohighlevel.com/dashboard/).
2. **Access the Agency Settings:**
   - From the dashboard, go to **Settings**.
   - Select **API Keys** from the side menu.
   - Select **Agency API Keys**.
   - Click on **Generate API Key** if you do not already have one.

> **Note:** Treat the `GoHighLevel Location API Key` as sensitive data. In SmythOS, you can securely manage it using the `Vault` feature. For more details, please refer to [this section](https://app.smythos.com/doc/#/vault/keysManagement) in the documentation.

## Get Contacts
Retrieve all contacts.

### Component Settings
To use this component:
- Click on the component settings.
- Add your `GoHighLevel Location API Key` from the `vault`. Please refer to the [Integration Settings](#integration-settings) to learn how to create it.
- Confirm and close the settings.

### Inputs
To effectively utilize this component, you need to provide the following inputs:
1. **trigger (`required<any value>`):** This input is required to trigger the execution of the component; it can hold any value.
2. **limit (`optional<integer>`):** This input sets the maximum number of contacts to retrieve. The default value is `20`.

> **Note:** With the flexibility and customization in mind, `SmythOS` allows you to expand the default input list by adding new inputs using the `+` button situated in the `Inputs` section.

### Outputs
After running this component, the following default outputs are generated:
1. **Response:** The raw response from the GoHighLevel API.
2. **Headers:** HTTP headers detailing the API request-response process.
3. **contacts:** The list of retrieved contacts.

> **Note:** Similar to inputs, you can add new outputs by clicking on the `+` button in the `Outputs` section. This feature can come in handy when you aim to extract specific values from the `Response`. Please refer [here](https://app.smythos.com/doc/#/components/APICall?id=example-without-authentication) to see a practical example.

## Create Contact
Creates a new contact.

### Component Settings
To use this component:
- Click on the component settings.
- Add your `GoHighLevel Location API Key` from the `vault`. Please refer to the [Integration Settings](#integration-settings) to learn how to create it.
- Confirm and close the settings.

### Inputs
To effectively utilize this component, you need to provide the following inputs:
1. **email (`required<string>`):** The email address of the contact.
2. **firstName (`optional<string>`):** The first name of the contact. Default is `N/A`.
3. **lastName (`optional<string>`):** The last name of the contact. Default is `N/A`.
4. **name (`optional<string>`):** The full name of the contact. Default is `N/A`.
5. **dateOfBirth (`optional<string>`):** The date of birth of the contact. Default is `N/A`.
6. **address1 (`optional<string>`):** The primary address of the contact. Default is `N/A`.
7. **city (`optional<string>`):** The city of the contact. Default is `N/A`.
8. **state (`optional<string>`):** The state of the contact. Default is `N/A`.
9. **postalCode (`optional<string>`):** The postal code of the contact. Default is `N/A`.
10. **companyName (`optional<string>`):** The company name of the contact. Default is `N/A`.
11. **website (`optional<string>`):** The website of the contact. Default is `N/A`.
12. **tags (`optional<array>`):** Tags associated with the contact. Default is `[]`.
13. **customField (`optional<object>`):** Additional custom fields for the contact. Default is `{}`.
14. **phone (`optional<string>`):** The phone of the contact. Default is `N/A`.

> **Note:** With the flexibility and customization in mind, `SmythOS` allows you to expand the default input list by adding new inputs using the `+` button situated in the `Inputs` section.

### Outputs
After running this component, the following default outputs are generated:
1. **Response:** The raw response from the GoHighLevel API.
2. **Headers:** HTTP headers detailing the API request-response process.
3. **id:** The ID of the newly created contact.
4. **locationId:** The location ID associated with the contact.
5. **email:** The email address of the contact.

> **Note:** Similar to inputs, you can add new outputs by clicking on the `+` button in the `Outputs` section. This feature can come in handy when you aim to extract specific values from the `Response`. Please refer [here](https://app.smythos.com/doc/#/components/APICall?id=example-without-authentication) to see a practical example.
