
# Google Drive Integration

`Google Drive` is a powerful cloud-based storage and synchronization service that offers users the ability to save files online, access them from anywhere, and collaborate with others seamlessly. Integrating `Google Drive` into `SmythOS` leverages these capabilities directly within the operating system, enhancing the functionality and efficiency of the workflow.

With `Google Drive` integration in `SmythOS`, users can enjoy direct access to their stored files and folders through the operating system's environment without the need to use a web browser or separate applications. This seamless integration empowers users to manage, edit, and share their files directly within `SmythOS`, making it an essential feature for professionals and teams who require real-time collaboration and flexibility in managing their digital assets.

In essence, `Google Drive` integration within `SmythOS` is not just about providing storage solutions, but rather enhancing the overall user experience by offering a unified, secure, and efficient way to manage digital content.

## Get Drive

This component retrieves the metadata of a specified shared drive in `Google Drive` through `SmythOS`.

### Settings

#### Client ID & Client Secret

For a visual step-by-step guide, watch this [video](#google-oauth-setup).

To connect `Google Drive` with `SmythOS`, you need to secure a `Client ID` and `Client Secret`. Following these comprehensive steps will guide you through the process of obtaining these credentials:

1. **Visit the Google Developers Console**:
   - Open the [Google Developers Console](https://console.developers.google.com/).
   - Start a new project or select an appropriate existing project for your `Google Drive` integration.

2. **Enable the Google Drive API**:
   - In the Developers Console, navigate to the `Library` section.
   - Locate and activate the `Google Drive API` for your project.

3. **Configure the OAuth Consent Screen**:
   - Update necessary details such as your application's name, email address, and needed scopes on the consent screen.
   - These settings are crucial for user permissions and secure access.

4. **Create Credentials**:
   - Go to the `Credentials` tab within the Developers Console.
   - Click on `Create credentials` and then choose `OAuth client ID`.
   - Specify your application's type and ensure to input all required redirects and configurations.
   - Set the callback URL to `https://app.smythos.com/oauth/google/callback` to handle proper redirection after users authenticate.

5. **Retrieve Client ID and Client Secret**:
   - After setting up the OAuth client ID, the console will provide you with your specific `Client ID` and `Client Secret`.

6. **Enter and Authenticate**:
   - Input the newly obtained `Client ID` and `Client Secret` into the designated fields in the settings section of the Google Drive component within `SmythOS`.
   - Hit the `Authenticate` button to secure the connection and begin utilizing `Google Drive` capabilities within `SmythOS`.

> **Note:** Treat the `Client ID` and `Client Secret` as sensitive data. In `SmythOS`, you can securely manage it using the `Vault` feature. For more details, please refer to [this](https://app.smythos.com/doc/#/vault/keysManagement) section in the documentation.

<Image src="/img/google_drive/screenshot4.png" alt="alt text" /> 

### Inputs

To retrieve drive metadata, configure the following input:

- **driveId (`required<string>`)**: The unique identifier of the drive whose metadata you wish to access.

<Image src="/img/google_drive/screenshot5.png" alt="alt text" />

> **Note:** Like many components of `SmythOS`, you can extend this list by adding new inputs with the `+` button in the `Inputs` section.

### Outputs

After successfully querying the drive, the endpoint provides detailed response data:

- **Response**: Comprehensive API response including all metadata of the queried drive.
- **Headers**: HTTP headers returned by the `Google Drive` API.
- **name**: Name of the drive.
- **kind**: Type of the resource (should return 'drive#drive').
- **id**: Unique identifier of the drive.

<Image src="/img/google_drive/screenshot6.png" alt="alt text" />

> **Note:** Similar to inputs, you can add new outputs by clicking on the `+` button in the `Outputs` section. This feature can come in handy when you aim to extract specific values from the `Response`. Please refer [here](https://app.smythos.com/doc/#/components/APICall?id=example-without-authentication) to see a practical example.
