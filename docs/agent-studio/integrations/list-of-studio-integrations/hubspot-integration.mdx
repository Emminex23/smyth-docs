
# HubSpot Integration

`HubSpot` is a leading CRM platform that helps companies attract visitors, convert leads, and close customers. It features a suite of tools to assist in marketing, sales, and service management, making it an essential asset for business growth and customer relationships management.

At `SmythOS`, we've seamlessly integrated with `HubSpot`, providing custom components that simplify the incorporation of its powerful services into your applications.

## Integration Settings

#### HubSpot Access Token
To use HubSpot integration in `SmythOs`, you must authenticate each integration component with a HubSpot Access Token. Follow these steps to prepare your credentials:
1. Log into your `HubSpot` account and navigate to the `Settings` menu.
2. Under '`Integrations`', go to `API Key`.
3. If an `API key` has not been generated, select '`Create Key`'. If it already exists, copy your existing `API key`.

> **Note:** Treat the `Access Token` as sensitive data. In `SmythOS`, you can securely manage it using the `Vault` feature. For more details, please refer to [this](https://app.smythos.com/doc/#/vault/keysManagement) section in the documentation.

## Create Contact
The `Create Contact` component in SmythOS facilitates the easy creation of contact records in `HubSpot`, enabling you to efficiently add new contacts into your CRM directly from your application.

### Component Settings
To use this component:
- Click on the component settings.
- Add your `HubSpot Access Token` from the `vault`. Please refer to the [Integration Settings](#integration-settings) to learn how to create it.
- Confirm and close the settings.

### Inputs
To effectively utilize this component, provide the following necessary information:
1. **email `required<string>`:** The email address of the contact.
2. **firstName `required<string>`:** The first name of the contact.
3. **lastName `required<string>`:** The last name of the contact.
4. **phone `required<string>`:** The phone number of the contact.
5. **company `required<string>`:** The company with which the contact is associated.
6. **website `required<string>`:** The website URL associated with the contact or company.

> **Note:** With the flexibility and customization in mind, `SmythOS` allows you to expand the default input list by adding new inputs using the `+` button situated in the `Inputs` section.

### Outputs
Upon execution of this component, you will receive the following outputs:
1. **Response:** Contains detailed information about the creation of the contact, including success or error messages.
2. **Headers:** The HTTP headers from the HubSpot server that provide additional metadata.
3. **id:** The unique identifier of the newly created contact.

> **Note:** Similar to inputs, you can add new outputs by clicking on the `+` button in the `Outputs` section. This feature can come in handy when you aim to extract specific values from the `Response`. Please refer [here](https://app.smythos.com/doc/#/components/APICall?id=example-without-authentication) to see a practical example.

## Import File
The `Import File` component facilitates the importation of files into the `HubSpot` file manager, ensuring seamless integration and management within your `HubSpot` account.

### Component Settings
To use this component:
- Click on the component settings.
- Add your `HubSpot Access Token` from the `vault`. Please refer to the [Integration Settings](#integration-settings) to learn how to create it.
- Confirm and close the settings.

#### Access
Define the accessibility and indexability of the imported file within `HubSpot`:
- `PUBLIC_INDEXABLE`: Publicly accessible and indexable.
- `PUBLIC_NOT_INDEXABLE`: Publicly accessible but not indexable.
- `HIDDEN_INDEXABLE`: Hidden from public view but indexable.
- `HIDDEN_NOT_INDEXABLE`: Hidden from public view and not indexable.
- `HIDDEN_PRIVATE`: Hidden and private.
- `PRIVATE`: Access restricted to certain users.

### Inputs
To ensure optimal performance, configure the following input fields accurately:
- **file_url (`required<string>`):** URL where the file is currently hosted.
- **folderPath (`required<string>`):** Path within `HubSpot` where the file will be stored.
- **name (`required<string>`):** Name to assign to the file in `HubSpot`.

> **Note:** With the flexibility and customization in mind, `SmythOS` allows you to expand the default input list by adding new inputs using the `+` button situated in the `Inputs` section.

1. **Response:** Provides a full response from `HubSpot`, including status and file details.
2. **Headers:** Meta-data and contextual information extracted from `HubSpot`'s response headers.
3. **id:** The unique identifier allocated to the file within `HubSpot`.

> **Note:** Similar to inputs, you can add new outputs by clicking on the `+` button in the `Outputs` section. This feature can come in handy when you aim to extract specific values from the `Response`.
