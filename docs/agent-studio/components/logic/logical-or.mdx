# Logic OR

The `Logic OR` component performs logical `OR` operations within your agents — ideal when **at least one** input needs to be present to trigger a condition.

---

## Definition and Rules

### Operational Rules

The component follows these rules:

| Input 1          | Input 2          | Verified   | Unverified |
|------------------|------------------|------------|------------|
| `Value`          | `Value`          | `True`     | `No Value` |
| `Value`          | `No Value/False` | `True`     | `No Value` |
| `No Value/False` | `Value`          | `True`     | `No Value` |
| `No Value/False` | `No Value/False` | `No Value` | `True`     |

> **Note:** `Verified` is `True` if **any** input has a value.  
> `Unverified` is `True` only when **all** inputs are missing or explicitly false.

---

## Inputs

### Boolean Values

- **True**: Any non-empty value.
- **False**: Explicit `false` or no value provided.

### ➕ Add Input

- **Name (Required):** A unique and descriptive name.
- **Type:** Choose from: Any, String, Number, Integer, Boolean, Array, Object, Binary.
- **Color (Optional):** Optional for visual organization.
- **Optional (Optional):** Mark if the input isn’t required.
- **Default Value (Optional):** Fallback value if nothing is passed.

---

## Outputs

1. **Verified:** Returns `True` if **any input** has value. Otherwise, returns `No Value`.
2. **Unverified:** Logical inverse of `Verified`. Returns `True` when **no inputs** have value.
