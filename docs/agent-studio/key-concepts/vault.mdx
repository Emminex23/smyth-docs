# Vault

The **Vault** in SmythOS is your secure hub for storing and managing sensitive credentials—like API keys, access tokens, and enterprise model configurations. All integrations across your agents pull from here, so nothing sensitive is hardcoded.

<InfoCallout>
Use the Vault to centralize authentication across all agents and integrations, including API requests, LLMs, webhooks, and automation tools.
</InfoCallout>

<Spacer size="md" />

## Built-In AI Providers

Use one of the SmythOS-managed OpenAI, Anthropic, Google AI, or Perplexity models with no extra setup. This enables you to build and test AI functionality quickly. When your agent's ready for production, configure and replace with your custom API key.

| Provider     | Availability        | Reference Link                                 |
|--------------|---------------------|-------------------------------------------------|
| OpenAI       | Built-in            | [OpenAI API](https://platform.openai.com/)      |
| Anthropic    | Built-in            | [Anthropic API](https://docs.anthropic.com/)    |
| Google AI    | Built-in            | [Google AI Console](https://console.developers.google.com) |
| Perplexity   | Built-in            | [Perplexity API](https://docs.perplexity.ai/)   |

<WarningCallout>
Built-in providers consume SmythOS credits (free credits included in [plans](/docs/account-management/subscription-management).). To use your own provider, add your custom keys.
</WarningCallout>

## Configuring Enterprise Models

To set up your own enterprise-grade AI model:

1. **Go to Vault** → Click `Add Enterprise Model`
2. Choose a provider: **Bedrock** or **Vertex AI**
3. Name your model and enable desired features (e.g., Text Completion)
4. Provide provider-specific credentials:

#### Provider Credential Requirements & Advanced Settings

| Provider         | Required Credentials                              | Advanced Settings                                      |
|-----------------|---------------------------------------------------|--------------------------------------------------------|
| Google Vertex AI | - Google Service Account Key  <br> - Project ID  <br> - Region | - Context window size  <br> - Max output tokens  <br> - Temperature  <br> - Top-p sampling  <br> - Custom response formatting |
| Amazon Bedrock   | - AWS Access Key ID  <br> - AWS Secret Access Key  <br> - Region | - Context window size  <br> - Max output tokens  <br> - Temperature  <br> - Top-p sampling  <br> - Custom response formatting |

## Manage API Keys

In the **API Keys** section, you can add and manage reusable tokens.

### Adding a Key

Each key requires:

| Field     | Description                             |
|-----------|-----------------------------------------|
| Key       | The API token (max 10,000 characters)   |
| Name      | A unique, recognizable label            |
| Scope     | Defines where the key can be used       |

### Scope Options

- `All` – No restriction  
- `API Call` – For [API Output](/docs/agent-studio/components/base/api-output)  
- `Hugging Face` – For [Hugging Face Integrations](/docs/agent-studio/integrations/hugging-face-integration)  
- `Zapier Action` – For [Zapier Workflows](/docs/agent-studio/integrations/zapier-integration)

## Use Vault Keys in Agents

SmythOS automatically injects keys from the Vault into agent components when scoped correctly.

### In [API Output Component](/docs/agent-studio/components/base/api-output)

- **Headers:** Add as `Authorization` or custom header  
- **URL Params / Body:** Inject directly or through mappings  
- **OAuth Credentials:** Use stored keys for client credentials

<Spacer size="md" />

### In [Hugging Face Integrations](/docs/agent-studio/integrations/hugging-face-integration)

- Paste or reference the Hugging Face token in the `Access Token` field.

### In [Zapier Actions](/docs/agent-studio/integrations/zapier-integration)

- Keys with `Zapier` scope will be auto-used by Zapier-triggered components.
